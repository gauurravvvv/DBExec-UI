Prompt Type Configuration Dialogs - Implementation Plan
Goal
Create configuration dialogs for each prompt type allowing users to configure:

Placeholder text - Hint text shown when empty
Default value - Pre-selected value(s)
Filter options - Enable/disable search filter
Filter label - Custom placeholder for filter input
Type-specific settings - Unique options per prompt type

Integration Points
1. Config Prompt Page
Add "Configure Display" button next to prompt type dropdown
Button visible only when prompt type is selected
Opens PromptConfigDialogComponent with current type
2. Data Storage
Store config as JSON in promptDisplayConfig field
Save with prompt configuration API
3. Execute Screen
Read config when rendering prompts
Apply placeholder, default values, filter settings
Implementation Steps
 Create PromptConfigInterface with all config types
 Create PromptConfigDialogComponent
 Implement dynamic form rendering based on type

 Add "Configure Display" button to config-prompt
