# 29 · AI Insights — Deep Test Cases

> All test cases assume the AI Insights surface from
> `docs/research/modules/25-ai-insights.md` is shipping.

## Fixtures
- Org with `enableAI=true`
- LLM provider configured (OpenAI / Ollama)
- Semantic model `orders` with `revenue`, `region`, `sold_at` metrics/dims.

## Happy
- **AI-H-01** · "revenue by region" produces a bar chart on right model. P0
- **AI-H-02** · Follow-up "for APAC only" adds a filter. P0
- **AI-H-03** · "summarise this visual" returns 2 sentences. P1
- **AI-H-04** · Source citation attached ("based on dataset `orders`"). P1
- **AI-H-05** · Confidence indicator shown. P1
- **AI-H-06** · Suggested next-question chips. P2
- **AI-H-07** · Anomaly callouts highlighted automatically. P1

## Negative
- **AI-N-01** · Request for undefined metric → "I don't have that". P0
- **AI-N-02** · Request that would expose masked column → refused. P0 🟣
- **AI-N-03** · Cross-org metric reference → refused. P0 🟣
- **AI-INJ-N-01** · Prompt-injection attempt (`Ignore previous instructions...`) → ignored. P0 🟣

## Edge
- **AI-CONV-H-01** · 5-turn conversation reuses context. P1
- **AI-PRIV-H-01** · Schema sanitisation: PII column names redacted before sending to LLM. P0 🟣
- **AI-COST-H-01** · Token usage recorded per org. P1
- **AI-COST-N-01** · Org over quota → rejected before LLM call. P1

## Audit
- **AI-AUD-H-01** · Each AI query writes audit row category=`ai_query`. P0
- **AI-AUD-N-01** · Sensitive tokens not stored. P0 🟣

## Security
- **AI-S-01** · No row-level data sent to LLM. P0 🟣
- **AI-S-02** · System prompt server-only; user input cannot reach tool descriptors. P0 🟣

## Performance
- **AI-P-01** · Streaming SSE first byte < 500ms. P1 ⚡
- **AI-P-02** · 30-token/s sustained streaming. P2 ⚡

## Regression buckets
- LLM integration → AI-H-01..04, AI-PRIV-H-01
- Privacy / injection → AI-INJ-N-01, AI-PRIV-H-01, AI-S-01..02
