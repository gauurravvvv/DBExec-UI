# 23 · Internationalization & Accessibility

> Two adjacent concerns: making DBExec usable in non-English
> locales (text, numbers, dates, currency, RTL) and making it
> usable for people who don't navigate with a mouse (keyboard,
> screen readers, sufficient contrast, reduced motion).
>
> Combined into one module because the same surfaces are touched
> (every component, every page, every error message). Skipping
> either at design time and bolting it on later costs ~10× more
> than weaving it in.
>
> Sister modules: [20 · Branding](20-branding.md) (contrast +
> theme-aware), [16 · Notifications](16-notifications.md) (locale
> in email templates).

**Depends on:** Angular i18n / @ngx-translate, Angular CDK, all FE modules
**Unblocks:** Non-English customers, EU/UK accessibility-compliant
  deployments, government contracts requiring WCAG AA
**Maturity:** 🟡 i18n keys exist (10 locales); a11y is patchy

---

**The patterns to copy:**

- **Locale = language + region** (e.g. `pt-BR`, `en-GB`). Not just
  language. Date / number / currency separator rules differ across
  regions of the same language.
- **ICU MessageFormat** for plural / gender / count: "{count, plural,
  one {1 visual} other {# visuals}}". Hand-rolled "1 visual / N
  visuals" doesn't translate to languages with multiple plural forms
  (Russian: 1, 2-4, 5+; Arabic: 6 forms).
- **CI guard** for raw English strings — every visible text MUST be
  in a translation key. The guard is a regex over the FE source.
- **WCAG AA at minimum**: 4.5:1 contrast for text, 3:1 for non-text,
  keyboard-only navigation everywhere, ARIA roles + names.
- **prefers-reduced-motion** respected at the CSS level — animations
  drop to 0 duration. Chart transitions disabled too.
- **Locale switching** at runtime — no page reload. A change to
  Spanish should update labels + dates + numbers without a refresh.

## 2. DBExec today

- 10 locale JSONs in `src/assets/i18n/*.json` (en, es, fr, de, ja,
  zh, pt, ru, ar, hi).
- `@ngx-translate/core` already integrated; most components consume
  translation keys.
- Coverage is patchy: ~30% of FE has hard-coded English strings.
- Numbers formatted via `Intl.NumberFormat`, but currency and
  percentage formatting is inconsistent.
- A11y: keyboard navigation works for the sidebar but breaks in
  the dashboard view and visual builder. No focus-trap on dialogs.
  Screen reader testing has never been done.
- No RTL support. CSS uses physical `left/right` everywhere.
- `prefers-reduced-motion` ignored.

### 4.2 ICU MessageFormat

Replace `@ngx-translate/core` text interpolation with
`@ngx-translate/messageformat-compiler` so plurals and gender
work correctly across languages:

```ts
// app.module.ts
import { TranslateMessageFormatCompiler } from 'ngx-translate-messageformat-compiler';

TranslateModule.forRoot({
  loader: { provide: TranslateLoader, useClass: HttpLoader },
  compiler: { provide: TranslateCompiler, useClass: TranslateMessageFormatCompiler },
})
```

JSON:

```json
{
  "dataset.list.count": "{count, plural, =0 {No datasets} one {1 dataset} other {# datasets}}",
  "user.invited_by": "{gender, select, female {She} male {He} other {They}} invited you."
}
```

Russian and Arabic plural rules work out of the box because ICU
handles them via Unicode CLDR.

### 4.3 Format helpers

```ts
// src/app/shared/services/format.service.ts
@Injectable({ providedIn: 'root' })
export class FormatService {
  private locale = signal<string>(localStorage.getItem('locale') ?? this.detectBrowserLocale());

  setLocale(loc: string) {
    this.locale.set(loc);
    localStorage.setItem('locale', loc);
    document.documentElement.lang = loc;
    document.documentElement.dir = this.isRtl(loc) ? 'rtl' : 'ltr';
  }
  getLocale() { return this.locale(); }

  formatNumber(n: number, options: Intl.NumberFormatOptions = {}): string {
    return new Intl.NumberFormat(this.locale(), options).format(n);
  }

  formatCurrency(n: number, currency: string): string {
    return new Intl.NumberFormat(this.locale(), { style: 'currency', currency }).format(n);
  }

  formatPercent(n: number, opts: { fractionDigits?: number } = {}): string {
    return new Intl.NumberFormat(this.locale(), {
      style: 'percent',
      minimumFractionDigits: opts.fractionDigits ?? 0,
      maximumFractionDigits: opts.fractionDigits ?? 2,
    }).format(n);
  }

  formatDate(d: Date | string, style: 'short'|'medium'|'long'|'full' = 'medium'): string {
    const date = typeof d === 'string' ? new Date(d) : d;
    return new Intl.DateTimeFormat(this.locale(), { dateStyle: style }).format(date);
  }

  formatRelative(d: Date | string): string {
    const date = typeof d === 'string' ? new Date(d) : d;
    const diffSec = (date.getTime() - Date.now()) / 1000;
    const rtf = new Intl.RelativeTimeFormat(this.locale(), { numeric: 'auto' });
    if (Math.abs(diffSec) < 60)       return rtf.format(Math.round(diffSec), 'second');
    if (Math.abs(diffSec) < 3600)     return rtf.format(Math.round(diffSec/60), 'minute');
    if (Math.abs(diffSec) < 86400)    return rtf.format(Math.round(diffSec/3600), 'hour');
    if (Math.abs(diffSec) < 86400*30) return rtf.format(Math.round(diffSec/86400), 'day');
    if (Math.abs(diffSec) < 86400*365) return rtf.format(Math.round(diffSec/(86400*30)), 'month');
    return rtf.format(Math.round(diffSec/(86400*365)), 'year');
  }

  formatFileSize(bytes: number): string {
    const units = ['B','KB','MB','GB','TB'];
    let i = 0;
    while (bytes >= 1024 && i < units.length - 1) { bytes /= 1024; i++; }
    return `${this.formatNumber(bytes, { maximumFractionDigits: 1 })} ${units[i]}`;
  }

  private isRtl(locale: string): boolean {
    return ['ar','he','fa','ur'].includes(locale.split('-')[0]);
  }

  private detectBrowserLocale(): string {
    const supported = ['en','es','fr','de','ja','zh','pt','ru','ar','hi'];
    const navLang = (navigator.language ?? 'en').toLowerCase();
    return supported.find(l => navLang.startsWith(l)) ?? 'en';
  }
}
```

### 4.4 RTL CSS — logical properties

Replace every physical property with its logical equivalent:

```css
/* Before (LTR-only) */
.card {
  padding-left: 16px;
  margin-right: 8px;
  border-left: 1px solid #ddd;
  text-align: left;
}

/* After (direction-aware) */
.card {
  padding-inline-start: 16px;
  margin-inline-end: 8px;
  border-inline-start: 1px solid #ddd;
  text-align: start;
}
```

Mapping:

```
left   → inline-start
right  → inline-end
top    → block-start
bottom → block-end
width  → inline-size
height → block-size
left:0 right:0  →  inset-inline: 0
```

Tailwind / Angular Material both support these as utility classes
(`ps-4`, `pe-2`, `ms-auto`, etc.).

For icons that are direction-sensitive (chevrons, arrows):

```css
.chevron-next {
  transform: scaleX(1);
}
[dir="rtl"] .chevron-next {
  transform: scaleX(-1);
}
```

### 4.6 CI guard for raw English

A pre-commit + CI check that fails on hard-coded strings in
component templates:

```bash
# scripts/check-i18n.sh
#!/usr/bin/env bash
# Find <button>, <span>, <p>, etc. with literal English text
# outside of {{ translate keys }}

FAIL=0

for f in $(find src/app -name "*.html"); do
  # Look for "<tag>English Text</tag>" patterns that aren't
  # interpolations and aren't pipes to translate
  matches=$(grep -nE \
    '>\s*[A-Z][a-zA-Z]{3,}[^<{]+<' "$f" \
    | grep -v 'translate' \
    | grep -v '\[innerHTML\]' || true)
  if [ -n "$matches" ]; then
    echo "❌ Raw English in $f:"
    echo "$matches"
    FAIL=1
  fi
done

if [ $FAIL -eq 1 ]; then
  echo ""
  echo "🚫 Hard-coded English strings detected."
  echo "   Move text into src/assets/i18n/*.json and use {{ 'key' | translate }}."
  exit 1
fi
```

Add to `husky` pre-commit + CI workflow.

There are inevitable allowlist exceptions (developer-facing
debug toolbar, etc.); use HTML comments to opt out:

```html
<!-- i18n-ignore: developer debug only -->
<span class="debug-pill">debug mode</span>
```

The checker treats `i18n-ignore` lines as suppressed.

### 4.7 Translation memory + machine translation

When a new feature adds keys to `en.json`, fan out to other
locales via:

```bash
# scripts/sync-translations.sh
#!/usr/bin/env bash
# 1. Diff en.json against es.json, fr.json, etc. — find missing keys
# 2. Call OpenAI / DeepL API to get a first-pass translation
# 3. Write to <locale>.json with a [MT] marker so human reviewers
#    can find machine-translated text later.

LOCALES="es fr de ja zh pt ru ar hi"
EN_FILE="src/assets/i18n/en.json"

for loc in $LOCALES; do
  LOC_FILE="src/assets/i18n/${loc}.json"
  python scripts/translate-diff.py \
    --source "$EN_FILE" --target "$LOC_FILE" \
    --provider deepl --mark-mt
done
```

The `[MT]` prefix gets stripped at runtime so the user doesn't see
it; the FE shows a "machine-translated" badge in non-prod builds so
reviewers know what to look at.

### 4.8 Per-org default locale

```sql
ALTER TABLE organisation
  ADD COLUMN default_locale varchar(8) NOT NULL DEFAULT 'en';
```

Resolution cascade at login:

```
user.locale  (per-user preference)
  → org.default_locale (org-level override)
    → request.acceptLanguage (browser hint)
      → 'en'
```

Server-side, the locale is stamped onto `res.locals.locale` by
middleware. Email and PDF rendering pick it up.

### 4.9 i18n in email + PDF

Email templates per locale:

```
src/shared/services/email/templates/
  subscription/
    en.hbs
    es.hbs
    fr.hbs
    ...
```

Rendering picks by recipient's user.locale or falls back to org's
default_locale. Same shape for PDF templates.

```ts
// src/shared/services/email/render.ts
async function renderEmail(recipient: User, template: string, data: any) {
  const locale = recipient.locale ?? recipient.organisation.defaultLocale ?? 'en';
  const tplPath = await resolveTemplate(template, locale);
  return Handlebars.compile(await readFile(tplPath, 'utf8'))(data);
}
```

### 4.10 i18n in API error responses

Server-side responses carry translation **keys**, not English text:

```json
{
  "status": false,
  "code": 400,
  "message": "validation.datasets.name.tooLong",
  "errors": [
    { "field": "name", "code": "string.max",
      "message": "validation.datasets.name.tooLong" }
  ]
}
```

The FE resolves the key against the user's locale before rendering:

```ts
// http error interceptor
intercept(req, next) {
  return next.handle(req).pipe(
    catchError(err => {
      if (err.error?.message?.startsWith('validation.') ||
          err.error?.message?.startsWith('errors.')) {
        const translated = this.translate.instant(err.error.message);
        this.toast.error(translated);
      } else {
        this.toast.error(err.error?.message ?? 'Unknown error');
      }
      return throwError(() => err);
    }),
  );
}
```

### 4.11 Keyboard navigation

Audit checklist applied to every component:

- Tab order matches visual order (no `tabindex="3"` hacks).
- Enter activates buttons. Space activates buttons + checkboxes.
- Arrow keys navigate inside lists / radios.
- Esc closes dialogs + popovers.
- Cmd-K opens search palette (module 17).
- `tabindex="0"` for custom interactive elements.
- `tabindex="-1"` for elements that can receive focus
  programmatically but aren't in tab order.

Focus-trap on dialogs (Angular CDK provides this):

```ts
import { FocusTrap, FocusTrapFactory } from '@angular/cdk/a11y';

@Component({ /* ... */ })
export class ShareDialogComponent {
  private trap?: FocusTrap;
  constructor(private focusTrapFactory: FocusTrapFactory,
              private elRef: ElementRef) {}

  ngAfterViewInit() {
    this.trap = this.focusTrapFactory.create(this.elRef.nativeElement);
    this.trap.focusFirstTabbableElementWhenReady();
  }

  ngOnDestroy() { this.trap?.destroy(); }

  @HostListener('keydown.escape')
  close() { /* ... */ }
}
```

### 4.12 ARIA

```html
<!-- Before -->
<div class="card" (click)="open()">
  <span class="icon">⚙</span>
  <span class="label">Settings</span>
</div>

<!-- After -->
<button class="card"
        type="button"
        (click)="open()"
        [attr.aria-label]="'settings.menu' | translate">
  <span class="icon" aria-hidden="true">⚙</span>
  <span class="label">{{ 'settings.label' | translate }}</span>
</button>
```

Rules:

- Use semantic HTML first (`<button>`, `<a>`, `<input>`).
- Add `role="button"` only to divs that genuinely can't be a
  `<button>` (rare).
- Every form input has a `<label for>` or `aria-labelledby` or
  `aria-label`.
- Decorative icons get `aria-hidden="true"`.
- Tooltips associate via `aria-describedby`.

### 4.13 Live-region announcements

For dynamic updates (chart finished loading, filter applied,
notification arrived):

```html
<!-- App-level sr-only live region -->
<div aria-live="polite" aria-atomic="true" class="sr-only" #liveRegion></div>
```

```ts
@Injectable({ providedIn: 'root' })
export class LiveAnnouncer {
  private liveRegion: HTMLElement | null = null;
  attach(el: HTMLElement) { this.liveRegion = el; }

  announce(message: string, priority: 'polite' | 'assertive' = 'polite') {
    if (!this.liveRegion) return;
    this.liveRegion.setAttribute('aria-live', priority);
    this.liveRegion.textContent = '';
    setTimeout(() => { if (this.liveRegion) this.liveRegion.textContent = message; }, 10);
  }
}

// Usage
this.liveAnnouncer.announce(
  this.translate.instant('dashboard.render.complete'),
);
```

### 4.15 High-contrast theme

Module 20's branding tokens accept a `high-contrast: BrandTokens`
preset that swaps:

- Backgrounds to pure black/white
- Borders to high-contrast (≥7:1)
- Hover/focus states with thick (3px) outlines
- Chart colours from the colourblind-safe palette

Users opt in via `prefers-contrast: more` media query OR a manual
toggle in user settings:

```ts
const wantsHighContrast =
  matchMedia('(prefers-contrast: more)').matches ||
  this.userPref.highContrast;
applyTokens(wantsHighContrast ? tokens.highContrast : tokens.light);
```

### 4.17 Chart alt-text / data-table fallback

Every chart has an accessible data-table sibling, exposed to
screen readers and hidden visually:

```html
<div class="visual">
  <div class="chart-container" aria-describedby="chart-data-table-{{ id }}">
    <!-- ECharts canvas -->
  </div>
  <table id="chart-data-table-{{ id }}" class="sr-only">
    <caption>{{ chart.title }}</caption>
    <thead>
      <tr>
        @for (col of columns; track col.name) {
          <th scope="col">{{ col.name }}</th>
        }
      </tr>
    </thead>
    <tbody>
      @for (row of rows; track $index) {
        <tr>
          @for (col of columns; track col.name) {
            <td>{{ row[col.name] }}</td>
          }
        </tr>
      }
    </tbody>
  </table>
</div>
```

Screen-reader users hear the chart's title + a navigable data
table. Visual users see the chart.

## 9. Migration & rollout

1. Phase 1 — i18n CI guard. New code MUST use translation keys.
   Existing English strings remain until they're refactored.
2. Phase 2 — FormatService + format pipes everywhere; deprecate
   ad-hoc `toFixed(2)`.
3. Phase 3 — ICU MessageFormat compiler integrated.
4. Phase 4 — Translation memory + machine-translate diff script.
5. Phase 5 — RTL CSS pass (logical properties).
6. Phase 6 — A11y triage: focus-trap, ARIA, axe-core CI.
7. Phase 7 — Live announcer + data-table fallback.
8. Phase 8 — High-contrast theme + colourblind palette.
9. Phase 9 — Per-org default locale + email/PDF locale resolution.

## 10. Open questions

- **Pseudolocale** for testing — automatic transformation of
  English to "Ëñĝłíšh" stretched 30% longer. Helps catch
  truncation bugs without translation. Use the `pseudo-localization`
  npm package.
- **Brand-name strings** — should "DBExec" itself be translated?
  No; keep as-is across all locales. Same for vendor names.
- **Plural in chart legends** — when a chart shows "5 series",
  must be ICU-pluralised. Easy to miss.
- **RTL with charts** — ECharts handles RTL text rendering;
  axis position flipping is FE's responsibility. Bar-chart-with-
  legend-on-the-right gets weird in RTL. Document.
- **Right-to-left filter dropdowns** — the calendar picker, range
  slider, and multi-select chips all need RTL audits.
- **Translation provider quality** — DeepL is best for European,
  Google Translate for Asian + Hindi, OpenAI for everything but
  expensive. Pipeline supports multiple, pick per-locale.
- **Locale negotiation in embed mode** — embed JWT can carry
  `locale: 'es'`; takes precedence over user's stored preference.
