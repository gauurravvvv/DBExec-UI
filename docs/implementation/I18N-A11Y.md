# i18n & accessibility

> Implementation companion to research module 23. Pins
> ICU MessageFormat plural/gender, FormatService for dates &
> numbers, RTL via CSS logical properties, a CI guard against
> raw English in templates, and axe-core CI for WCAG AA.

**Status:** 🔴 not in product.
**Effort:** L (~3 weeks the first pass; ongoing).

---

## 0. Problem statement

DBExec ships in English today. Customers want it in Spanish,
French, German, Portuguese, Japanese, Korean, Arabic (RTL),
Hebrew (RTL). Plus accessibility — WCAG 2.1 AA for keyboard
users, screen readers, low vision, motor impairment.

Both are *cross-cutting* concerns: every component needs a
discipline. The discipline is what this doc pins.

## 1. ICU MessageFormat

Plurals, gender, ordinals, selects:

```icu
// en
You have {n, plural,
  =0 {no notifications}
  one {one notification}
  other {# notifications}
}.

// fr
Vous avez {n, plural,
  =0 {aucune notification}
  one {une notification}
  other {# notifications}
}.

// ar — six plural forms, Unicode CLDR
لديك {n, plural,
  zero {…}
  one  {…}
  two  {…}
  few  {…}
  many {…}
  other {…}
} إشعار.
```

Library: `formatjs` (`@formatjs/intl-messageformat`). Angular
binds via `@formatjs/intl`-driven directive or via a tiny
adapter to Angular's existing i18n.

## 2. Catalogue format & flow

Source-of-truth: `src/locales/en.json`:

```json
{
  "dashboard.title": "Dashboard",
  "notification.count": "You have {n, plural, =0 {no notifications} one {one notification} other {# notifications}}.",
  "user.greeting": "Hello, {name}!"
}
```

Translation pipeline:

1. CI extracts every `t('key')` and `i18n` directive into
   `en.json`.
2. Push to translation vendor (Lokalise / Crowdin) or
   committed to git for community translation.
3. Pull translated `fr.json`, `es.json`, etc.
4. Build embeds JSON into bundles (per-locale chunks for
   tree-shaking).

## 3. FormatService

```typescript
@Injectable({ providedIn: 'root' })
export class FormatService {
  private readonly locale = signal<string>('en');
  private readonly tz = signal<string>('UTC');

  set(locale: string, tz: string): void { this.locale.set(locale); this.tz.set(tz); }

  number(value: number, opts?: Intl.NumberFormatOptions): string {
    return new Intl.NumberFormat(this.locale(), opts).format(value);
  }
  currency(value: number, currencyCode: string): string {
    return new Intl.NumberFormat(this.locale(),
      { style: 'currency', currency: currencyCode }).format(value);
  }
  percent(value: number, decimals = 1): string {
    return new Intl.NumberFormat(this.locale(),
      { style: 'percent', minimumFractionDigits: decimals, maximumFractionDigits: decimals }).format(value);
  }
  date(value: Date | string, opts?: Intl.DateTimeFormatOptions): string {
    const d = typeof value === 'string' ? new Date(value) : value;
    return new Intl.DateTimeFormat(this.locale(),
      { timeZone: this.tz(), ...opts }).format(d);
  }
  relative(value: Date | string): string {
    const d = typeof value === 'string' ? new Date(value) : value;
    const diff = (d.getTime() - Date.now()) / 1000;
    const rt = new Intl.RelativeTimeFormat(this.locale(), { numeric: 'auto' });
    if (Math.abs(diff) < 60) return rt.format(Math.round(diff), 'second');
    if (Math.abs(diff) < 3600) return rt.format(Math.round(diff / 60), 'minute');
    if (Math.abs(diff) < 86_400) return rt.format(Math.round(diff / 3600), 'hour');
    if (Math.abs(diff) < 30 * 86_400) return rt.format(Math.round(diff / 86_400), 'day');
    return this.date(d);
  }
}
```

Every visible number, date, currency in the app goes through
this service. Hard-coded `.toFixed(2)` is a code-review block.

## 4. RTL via logical properties

CSS uses **logical** properties instead of left/right /
top/bottom:

```css
/* WRONG */
.tile { padding-left: 12px; margin-right: 8px; border-left: 1px solid; }

/* RIGHT */
.tile {
  padding-inline-start: 12px;
  margin-inline-end: 8px;
  border-inline-start: 1px solid;
}
```

The HTML root gets `dir="rtl"` when the user's locale is RTL;
the browser flips all logical-property layouts automatically.

A `stylelint` rule blocks new `margin-left` / `padding-right`
etc. in PRs.

## 5. CI guard against raw English

```typescript
// scripts/scan-untranslated.ts
import { Project, SyntaxKind } from 'ts-morph';

const project = new Project({ tsConfigFilePath: './tsconfig.json' });
const issues: string[] = [];

for (const file of project.getSourceFiles('src/**/*.{html,ts}')) {
  // Naive but effective: any string literal of >= 3 letters that
  // isn't in an allowlist and isn't piped through t() or i18n
  const text = file.getText();
  const matches = text.matchAll(/>([A-Z][a-z]{2,}(?: [A-Z]?[a-z]+)+)</g);
  for (const m of matches) {
    if (!isAllowlisted(m[1])) issues.push(`${file.getFilePath()}: ${m[1]}`);
  }
}

if (issues.length) {
  console.error(`Found ${issues.length} untranslated strings:\n${issues.join('\n')}`);
  process.exit(1);
}
```

Run as a CI step. Allowlist for proper nouns, brand names,
SQL keywords.

## 6. WCAG 2.1 AA via axe-core

In Playwright e2e:

```typescript
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

test.describe('a11y', () => {
  for (const route of ['/login', '/app', '/app/dashboards/sample', '/app/analyses/sample']) {
    test(`${route} passes WCAG 2.1 AA`, async ({ page }) => {
      await page.goto(route);
      const results = await new AxeBuilder({ page })
        .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'])
        .analyze();
      expect(results.violations, JSON.stringify(results.violations, null, 2)).toHaveLength(0);
    });
  }
});
```

Per-component unit tests use `@axe-core/react`-equivalent for
Angular: render the component standalone, run axe, expect zero
violations.

## 7. Focus management

Modals trap focus, restore on close. A `focus-trap` directive:

```typescript
@Directive({ selector: '[appFocusTrap]', standalone: true })
export class FocusTrapDirective {
  private trap?: FocusTrap;
  constructor(private el: ElementRef<HTMLElement>) {}

  ngAfterViewInit() {
    this.trap = createFocusTrap(this.el.nativeElement, {
      escapeDeactivates: true,
      clickOutsideDeactivates: false,
      returnFocusOnDeactivate: true,
    });
    this.trap.activate();
  }
  ngOnDestroy() { this.trap?.deactivate(); }
}
```

Used on every modal, slide-over, dialog.

## 8. ARIA live regions for async updates

```html
<div aria-live="polite" aria-atomic="true" class="sr-only" *ngIf="status$ | async as s">
  {{ s.message }}
</div>
```

Use cases:
- "Loading dashboard..."
- "Export ready, click to download"
- "10 new notifications"

Without this, screen-reader users have no idea the UI changed.

## 10. High-contrast + colourblind palettes

## 11. Chart data-table fallback

Every chart has a hidden `<table>` with the same data, exposed
to screen readers via:

```html
<div class="chart-container">
  <app-echart-visual [option]="option"></app-echart-visual>
  <table class="sr-only">
    <caption>{{ chartTitle }} – data table</caption>
    <thead><tr><th>Region</th><th>Revenue</th></tr></thead>
    <tbody>
      <tr *ngFor="let r of rows"><td>{{ r.region }}</td><td>{{ format.currency(r.revenue, 'USD') }}</td></tr>
    </tbody>
  </table>
</div>
```

Visually hidden via `.sr-only` (positioned offscreen).
Screen readers read the table; sighted users see the chart.

## 12. Keyboard support

Every interactive element reachable via Tab. Custom widgets
(combobox, drag-drop) follow WAI-ARIA Authoring Practices:

- Combobox: Up/Down navigates options, Enter selects, Esc
  closes, Home/End jump.
- Drag-drop reorder: focus item, Space picks up, Arrow keys
  move, Space drops, Esc cancels.
- Chart cells: Tab into chart, Arrow keys navigate cells,
  Enter triggers configured action (action module).

## 14. Security & threat model

| Threat | Mitigation |
|---|---|
| Translation XSS (translator injects script) | All translations rendered as text; never `innerHTML`; CSP forbids inline scripts |
| Locale spoofing via header | Locale picked from user profile preference first, Accept-Language second; never from URL alone |
| ICU expression bomb | `formatjs` bounds; max nesting depth enforced at parse time |
| RTL homograph attack in user-content | display-only sanitisation strips bidi control chars from user-provided text |

## 15. Runbook

**Symptom: French dashboard missing translations.**
1. `dbexec_i18n_missing_key_total{locale='fr'}` rising.
2. Re-run extractor + re-import from vendor.

**Symptom: axe CI breaks on every PR.**
1. New component skipped a11y. Run `axe` locally; fix
   focus, ARIA, contrast.

**Symptom: customer reports "screen reader skipped a chart".**
1. Data-table fallback missing or empty. Verify component
   wired.

## 17. Migration & rollout

1. **Migrations:** `user.locale`, `user.timezone` columns; `org.default_locale`.
2. **Backfill:** all existing users default to `en` + UTC.
3. **Feature flag:** none — i18n + a11y ship together as a quality
   upgrade.
4. **First language pack:** Spanish + French + German. Then
   Japanese, Korean, Brazilian Portuguese.
5. **A11y enforcement:** axe CI starts as a *warning* check
   for two weeks while existing violations are fixed; then
   becomes blocking.
